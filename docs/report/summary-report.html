所感

今回、約2時間ほどで検証用チャットアプリの開発を他の業務と並行して行いました。
自分でコードは一切書かずに命令だけで、要件定義からテスト・GitHub管理まで全工程を完結させることができました。

この結果から、自身のリソースを最小限に抑えながら、複数のプロダクト開発を同時並行で進められる可能性を強く感じました。

一方で、今回は比較的シンプルなプロダクトでの検証だったため、難易度の高いプロダクトになれば相応に自身のリソースを割く必要が出てくるとも感じています。
ただ、それを加味しても生産性の向上は間違いなく大きいと確信しています。
