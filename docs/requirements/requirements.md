## 🔧 機能要件

### FR-001: チャット機能（コア）

- **優先度**: High
- **説明**: ユーザーはテキストを入力してAIと会話できる
- **受入基準**:
  - [ ] メッセージ入力フォームが表示される
  - [ ] 送信ボタンまたはEnterキーでメッセージを送信できる
  - [ ] AIの返答がリアルタイムまたは短時間で表示される
  - [ ] 会話履歴がチャット画面に表示される（ユーザー・AI交互）

### FR-002: チャット履歴管理

- **優先度**: Medium
- **説明**: セッション内の会話履歴を保持・表示する
- **受入基準**:
  - [ ] セッション中は会話履歴が保持される
  - [ ] 「会話をリセット」ボタンで履歴をクリアできる
  - [ ] スクロール可能なチャットビューで過去のメッセージを確認できる

### FR-003: レスポンシブUI

- **優先度**: Medium
- **説明**: PC・スマートフォンで快適に使用できるレイアウト
- **受入基準**:
  - [ ] モバイル（375px〜）からデスクトップ（1440px）まで正常表示
  - [ ] 入力エリアが画面下部に固定される

### FR-004: ローディング・エラー表示

- **優先度**: Medium
- **説明**: AI処理中・エラー時に適切なフィードバックを提供する
- **受入基準**:
  - [ ] メッセージ送信中はローディングインジケーターが表示される
  - [ ] APIエラー時はエラーメッセージが表示される

## 🛠 技術制約

### TC-001: 技術スタック

| 項目 | 指定バージョン |
|------|---------------|
| Node.js | 22.x (LTS) |
| TypeScript | 5.3+ (strict mode) |
| Next.js | 15+ (App Router) |
| Tailwind CSS | 3.x |
| openai | 最新版 |
| モデル | gpt-4o |

### TC-002: アーキテクチャ

- Next.js App Router構成
- チャットAPIはサーバーサイドで実装（`/api/chat`）
- クライアントコンポーネントは`"use client"`明示
- OpenAI APIキーは `.env.local` の `OPENAI_API_KEY` で管理

### TC-003: 禁止事項

- ❌ APIキーのクライアントサイド露出
- ❌ JavaScript（TypeScriptに統一）
