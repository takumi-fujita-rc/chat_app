# 認証機能 要件定義書

> Issue: #7
> 生成日: 2026-03-13

---

## 概要

| 項目 | 内容 |
|------|------|
| 機能名 | シンプルID/パスワード認証 |
| 目的 | チャット画面への不正アクセスを防止する |
| 認証方式 | NextAuth.js Credentials Provider（固定アカウント） |

---

## 機能要件

### FR-AUTH-001: ログイン画面
- メールアドレス＋パスワードの入力フォームを表示する
- 送信後、認証成功時はチャット画面へリダイレクト
- 認証失敗時はエラーメッセージを表示する

### FR-AUTH-002: アクセス制御
- 未認証ユーザーがチャット画面（`/`）にアクセスした場合、ログイン画面（`/login`）へリダイレクトする
- 認証済みユーザーがログイン画面にアクセスした場合、チャット画面へリダイレクトする

### FR-AUTH-003: ログアウト
- チャット画面のヘッダーにログアウトボタンを表示する
- ログアウト後はログイン画面へリダイレクトする

### FR-AUTH-004: セッション管理
- セッションはJWTで管理する
- セッション有効期限は24時間とする

---

## 非機能要件

| 項目 | 内容 |
|------|------|
| アカウント管理 | 環境変数で固定アカウントを管理（DB不要） |
| パスワード保護 | 環境変数に平文で保存（検証用途のため） |
| APIキー保護 | 認証APIルートはサーバーサイドのみ |

---

## 認証アカウント（検証用）

| 項目 | 値 |
|------|-----|
| メールアドレス | `admin@example.com` |
| パスワード | 環境変数 `AUTH_PASSWORD` で管理 |

---

## 画面遷移

```
未認証
  ↓ / にアクセス
/login（ログイン画面）
  ↓ 認証成功
/（チャット画面）
  ↓ ログアウト
/login（ログイン画面）
```
